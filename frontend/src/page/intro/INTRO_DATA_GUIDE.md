# How to fill in `departments.json` from Figma

**File location:** `frontend/src/page/intro/departments.json`

**Image location:** put files in `frontend/public/intro/` (banners and group photos) and `frontend/public/intro/activities/` (activity images). The path you write in the JSON starts with `/intro/...` — never include `public/` in the path.

---

## 1. Top-level structure

The file is one big object keyed by department slug:

```json
{
  "operation":  { ... },
  "publicity":  { ... },
  "technical":  { ... }
}
```

**Slug rules:** lowercase, ASCII letters only, no spaces. The slug becomes part of the URL (e.g. `/intro/operation`). Do not rename existing slugs — that will break links.

---

## 2. Per-department fields (from Figma 部門介紹 page)

| Field | Where to find it in Figma | Example |
|---|---|---|
| `title` | The Chinese department name in the banner / page heading | `"學術部"` |
| `bannerImage` | The wide colored banner at the top of the department page — export as JPG | `"/intro/technical-banner.jpg"` |
| `groupPhotoImage` | The 合照 image in the middle section — export as JPG | `"/intro/technical-group-photo.jpg"` |
| `introParagraphs` | The body text under "○○部簡介". Split into one string per paragraph | `["大家好…", "我們很希望…"]` |
| `memberDescription` | The text under 部員名單 | `"致力於服務系上同學…"` |
| `extraSidebarItems` | Any sidebar labels beyond 部門介紹 / 部門活動 (e.g. 學習能力, 體驗談). Leave `[]` if none | `[{ "label": "學習能力" }]` |

**Do not add 部門介紹 or 部門活動 to `extraSidebarItems`** — those are auto-generated by the page.

---

## 3. Activities (from Figma 部門活動 grid + each activity detail page)

Each department has an `activities` array. **The order matters** — it controls the order of cards on the activity grid, and the Prev / Next arrows on detail pages.

```json
"activities": [
  {
    "slug": "eesa-web",
    "name": "系學會網頁",
    "image": "/intro/activities/department-website.jpg",
    "description": ["負責系學會各類網站…"]
  }
]
```

| Field | Where to find it in Figma |
|---|---|
| `slug` | English URL-friendly name. Lowercase, hyphens only. **Cannot contain spaces or Chinese.** Must be unique within this department |
| `name` | The Chinese activity title shown on the card and detail page |
| `image` | The activity's photo on its detail page — export as JPG and save to `frontend/public/intro/activities/` |
| `description` | The body text on the activity detail page, split into one string per paragraph. Use 1 element if there is only one paragraph |

---

## 4. Adding a new activity (no code change needed)

1. Add a new object to the department's `activities` array
2. Save the activity's image into `frontend/public/intro/activities/` and use the matching path
3. The URL `/intro/{department-slug}/activities/{activity-slug}` will work automatically
4. The activity card will appear on `/intro/{department-slug}/activities` automatically
5. The Prev / Next links recompute based on array order — re-order the array to re-order the gallery

---

## 5. Adding a new department

1. Add a new top-level key in the JSON (e.g. `"activity"` for 活企部)
2. Fill in all the per-department fields described in section 2
3. Update `EesaIntro.jsx` department cards' `link` field to `/intro/{new-slug}` if it should be linked from the homepage

---

- `technical-banner.jpg`

**Group photos** (in `frontend/public/intro/`)

**Activity images** (in `frontend/public/intro/activities/`)

---

## 7. Common mistakes to avoid

- **JSON syntax**: every string in double quotes; commas between items but **not** after the last one in an array or object; no trailing commas
- **Slug spaces / Chinese**: URL slugs must be ASCII lowercase only — put Chinese in `name` or `title`, never in `slug`
- **Image paths**: must start with `/intro/...`, never `public/intro/...` or `./intro/...`
- **Paragraph splitting**: don't put `\n` inside one string for paragraph breaks — make each paragraph its own element of the array

## 8. Quick example: complete department entry

```json
"technical": {
  "title": "學術部",
  "bannerImage": "/intro/technical-banner.jpg",
  "groupPhotoImage": "/intro/technical-group-photo.jpg",
  "introParagraphs": [
    "大家好，我們是系學會的學術部…",
    "我們很希望可以透過自己幫助到系上的同學們…"
  ],
  "memberDescription": "致力於服務系上同學，舉辦各項活動並促進師生交流。組織包含會長與四大部門",
  "extraSidebarItems": [],
  "activities": [
    {
      "slug": "eesa-web",
      "name": "系學會網頁",
      "image": "/intro/activities/department-website.jpg",
      "description": [
        "負責系學會各類網站的前後端開發與維護…"
      ]
    }
  ]
}
```
